# Happy Star 星图皮肤 V4 视觉契约

日期：2026-07-07

## 1. 目标

V4 的目标不是再调整星图布局，而是把 `docs/prototypes/reference/star-map-target.png` 固化为视觉母版，将 V3 中“布局接近但材质粗糙”的问题拆成可验收的视觉指标。

母版信息：

- 文件：`docs/prototypes/reference/star-map-target.png`
- 尺寸：`1610 x 977`
- 来源：本项目会话中生成的 Happy Star 星图效果图

V4 仍是独立 HTML 原型，不接真实 API，不修改生产 React/Fastify 代码。通过 V4 验收后，后续 React 星图皮肤必须以 V4 为视觉契约。

## 2. 非目标

- 不把母版截图直接作为页面背景来“假装实现”。
- 不引入 Three.js、WebGL、Canvas 粒子库或重型 UI 库。
- 不修改现有积分、兑换、审批、产能算法。
- 不提交 `data/`。
- 不追求逐像素复刻，目标是视觉语言、材质和层级接近母版。

## 3. 总体视觉原则

V4 必须先像一个“星图能量仪表盘”，再像一个普通后台页面。布局可以沿用 V3，但视觉重点要从盒子布局转移到材质：

- 背景有远近星尘、少量星座线、微光颗粒和局部星云。
- 任务是发光徽章，不是圆形按钮。
- 奖励是紫色玻璃星球，带明暗面、星环和内部图标。
- 中心是温暖金色能量核心，文字清楚，不被亮光淹没。
- 面板是深色半透明玻璃，文字层级清晰。

## 4. 背景验收

母版特征：

- 深蓝黑背景，不是纯黑或简单渐变。
- 中央区域有青绿色星云薄雾。
- 全屏有不同亮度和大小的星点，分布不均匀。
- 有少量细线星座，线条低透明度，不能抢任务主线。
- 中央能量核心周围有金色环形网格和虚线刻度。

V4 要求：

- 使用 CSS/SVG 多层背景：基础渐变、星尘层、星座线层、中心星云层。
- 星点至少分 3 层：远景小点、中景亮点、少量十字高光星。
- 星座线只作为装饰，透明度低于任务连线。
- 背景不允许大面积黄色或紫色铺满。

## 5. 任务徽章验收

母版特征：

- 任务节点是半透明玻璃徽章，有内外两层圆环。
- 外层有虚线或细点轨道。
- 完成状态右上角有小圆对勾，颜色与任务主题一致。
- 节点内图标像定制图标，不像系统 emoji。
- 任务卡片和圆球互相贴近，像一个整体组件。

V4 要求：

- 使用 `.task-orb` 表示任务徽章。
- 每个任务徽章包含：外部辉光、虚线轨道、内层玻璃、高光、SVG 图标、完成徽章。
- 不再直接使用 emoji 作为主要图标；任务图标用内联 SVG 绘制。
- 颜色分别保持：作业粉、阅读蓝、卫生青绿、帮助紫。
- 任务完成点击后，任务徽章要出现短暂 `orb-pop` 和 `+N ⭐` 上浮。

## 6. 奖励星球验收

母版特征：

- 奖励星球是紫色玻璃球，不是平面圆。
- 球体有左上高光、右下暗部、内部纹理和外部辉光。
- 星环绕球体穿过前后，有前后遮挡关系。
- 星球内部是金币、书、星星等图标，图标本身也有光。

V4 要求：

- 使用 `.reward-planet` 表示奖励星球。
- 每个星球包含：背后星环、球体渐变、表面高光、暗部阴影、前景星环、内联 SVG 图标。
- 星环不能只是一个完整椭圆，必须通过前后两层制造穿插感。
- 奖励卡片仍显示名称、积分价格、预计到手时间。

## 7. 中心能量核心验收

母版特征：

- 中心是金色核心，内部有星形符号和清晰文字。
- 周围有多圈雷达环、虚线刻度和光点。
- 亮度温暖但不盖住文字。

V4 要求：

- 使用 `.energy-core` + `.radar-rings`。
- 星形图标与文字必须同时清楚。
- 中心核心亮度低于母版截图中最亮的按钮高光，避免 V3 那种白块感。
- 任务/奖励主连线从核心附近发出，线条有节点光点。

## 8. 面板与导航验收

母版特征：

- 左右面板是深色玻璃，不是纯黑卡片。
- 边框很细，内部有轻微青色反光。
- 文字用暖白、金色、粉色、蓝色分层。
- 顶部导航是胶囊形玻璃条，当前项有金色底部光线。

V4 要求：

- 面板背景使用 `rgba` + `backdrop-filter` + 内阴影。
- 面板边框透明度不超过任务卡片边框。
- 家长管理按钮用图标按钮，避免大块文字按钮。
- 720px 高度下不能裁切右侧卡片或底部命令栏。

## 9. 动效验收

V4 原型必须保留以下交互：

- 点击“帮助他人”任务后：
  - 分数从 `380` 增加到 `383`。
  - 帮助任务圆球出现完成对勾。
  - 对应连线出现 600-900ms 流光。
  - 出现 `+3 ⭐` 上浮。
  - 出现短提示。
- 点击“蛋币”奖励后：
  - 出现 `蛋币：480 ⭐，预计到手 3.7 周` 提示。
  - 奖励星球有短暂脉冲。

## 10. 实施策略

推荐策略：CSS 布局 + 内联 SVG 图标 + 少量 CSS 材质层。

原因：

- 保持文件独立，便于对比和提交。
- 不增加运行依赖。
- 能明显提升质感，不需要 WebGL。
- 后续迁移 React 时可拆成组件。

V4 文件：

- 新增：`docs/prototypes/star-map-skin-v4.html`
- 保留：`docs/prototypes/star-map-skin-v3.html`
- 参考：`docs/prototypes/reference/star-map-target.png`

## 11. 验证方式

必须执行：

- 浏览器打开 V4 原型并截图。
- 检查 console error 为 0。
- 在 1280x720 下检查不裁切。
- 点击帮助任务，验证分数、对勾、上浮、toast。
- 点击蛋币奖励，验证 toast。
- 静态扫描无第三方品牌、占位词、示例文本或外链残留。
- `npm test` 通过。
